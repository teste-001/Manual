Conceitos Básicos:

  • O que é controle de versão:
    • Definição: É como uma linha do tempo que registra as mudanças no código, facilitando a colaboração e o rastreamento de problemas.
    • Exemplos de sistemas: Softwares como Git, SVN e Mercurial.
  • Por que fazer controle de versão:
    • Benefícios: Organiza e salva versões do código para evitar perdas.
    • Importância: Ajuda equipes a trabalhar juntas e ver quem fez o quê.
  • O que é um repositório no Git:
    • Definição: É um local onde o Git armazena todas as versões do seu projeto.
    • Como funciona: O Git rastreia mudanças e permite que você volte a qualquer versão anterior.
  • Diferença entre repositórios do Git e GitHub:
    • Explicação: Git é o sistema que rastreia mudanças, enquanto GitHub é um site onde você pode armazenar projetos controlados pelo Git.

Parte da Programação:

  • Semântica e sintaxe:
    • Diferença: Semântica é o significado do código, enquanto a sintaxe é a forma como você escreve.
  • Backend e Frontend:
    • Distinção: Backend é o "cérebro" de um site (servidor, banco de dados), enquanto Frontend é o que você vê e interage (design, botões).

Comandos Simples:

  • Como criar um repositório usando comandos do Git:
    • Comando: git init
    • Descrição: Começa um novo projeto com Git.
  • Comandos Básicos:
    • Caso de case sensitive: O Git diferencia letras maiúsculas de minúsculas.
    • Control + L: Limpa a tela no terminal.
    • cd ..: Volta para a pasta anterior no terminal.
    • Mkdir: Cria uma nova pasta.
    • Pwd: Mostra o caminho da pasta atual no terminal.

Configurações Iniciais:

  • Configuração de email e usuário:
    • Descrição: Configura o email e nome do usuário para identificação no Git.
  • git init.defaultBranch:
    • Significado: Define o nome padrão para a branch principal ao iniciar um repositório.

Funcionalidades dentro do GitHub (github.com):

  • O que é README FILE:
    • Explicação: É um arquivo que descreve e informa sobre um projeto no GitHub.
  • PUBLIC VS PRIVATE:
    • Diferença: Repositórios públicos podem ser vistos por todos, enquanto os privados são restritos.

Processo de Conexão com o Servidor:

  • git remote add origin master URL:
    • Descrição: Conecta-se a um repositório remoto.
  • git remote --v:
    • Descrição: Lista os repositórios remotos configurados.
  • Push and Pull:
    • Descrição: Envia e recebe alterações de um servidor remoto.

Comandos de Apoio:

    • Descrição: Fornece ajuda sobre os comandos do Git.
  • Git help log:
    • Descrição: Fornece ajuda específica para o comando "log".

Sessão Erros:

  • Erro de merge:
    • Descrição: Um conflito ocorreu quando duas alterações não puderam ser combinadas automaticamente.
  • Fatal: This operation must be run in a work tree:
    • Descrição: O comando deve ser executado em um diretório de trabalho do Git.
